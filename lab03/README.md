### Клеточные автоматы. Лесные пожары (GUI)

**Задание:**  
Реализовать моделирование возникновения и распространения лесных пожаров с использованием двумерного клеточного автомата.

**Требования:**
- реализовать **не менее трёх дополнительных правил** поведения системы.

### Дополнительные правила:

**Влажность**

Она снижает вероятность того, что дерево загорится от горящей клетки в окрестности. 
В методе `get_ignition_chance` вычисляется базовый шанс распространения огня, 
зависящий от значения ползунка влажности в интерфейсе:

```python
spread_prob = 1.0 - self.humidity.get()

# проверка числом от 0 до 1
if random.random() < spread_prob:
    return True # клетка загорается
```

Если в интерфейсе влажность = 0.8, то `spread_prob` составит 0.2. 
Дерево загорится от соседа с шансом 20%.

**Направление ветра**

Правило заставляет пожар распространяться охотнее с указанного направления,
нежели против него. В цикле проверки восьми соседей программа анализирует 
смещения `dx` (верх/низ) и `dy` (лево/право) относительно текущей клетки:

```python
# если выбран "северный" ветер, то он дует с севера на юг
if wind == "Северный":
    if dx == -1: spread_prob *= 1.8  # сосед сверху поджигает быстрее (по ветру)
    if dx == 1:  spread_prob *= 0.1  # сосед снизу почти не может поджечь (против ветра)
elif wind == "Южный":
    if dx == 1:  spread_prob *= 1.8  # сосед снизу поджигает быстрее
    if dx == -1: spread_prob *= 0.1
# аналогично для восточного (dy=1) и западного (dy=-1) ветров
```

Благодаря этим множителям пожар принимает вытянутую форму эллипса.

**Длительность горения**

Обычно дерево сгорает за 1 такт. Данное правило вводит стадии горения, имитируя наличие
у дерева ограниченного запаса "топлива", который расходуется постепенно.
В методе `update_step` для клеток, которые уже горят (состояние 2 и выше), вводится счетчик 
циклов, который сравнивается с лимитом `max_burn` из интерфейса:

```python
elif state >= BURNING:
    # если дерево горит, увеличиваем стадию (значение клетки в массиве)
    if state < max_burn:
        new_grid[x, y] += 1
    else:
        # если время вышло, дерево превращается в 0 (пустоту)
        new_grid[x, y] = EMPTY
```

Пока значение клетки увеличивается, она остается опасной для соседей. Визуально это 
отображается сменой цвета: оранжевый цвет на начальном этапе
и темно-красный на последующих.
